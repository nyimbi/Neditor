# Performance Profile Runbook

Use this runbook for release-device native performance evidence.

## Scope

The profile must cover a sustained native session on a representative release
device, not only a unit test or browser mock.

## Required Scenarios

- Cold startup and first document render.
- Opening a large document.
- Sustained editing with debounced preview updates.
- Transform-heavy document compilation with cache reuse.
- Export suite execution.
- File-watch reload and conflict handling.
- Agent Workspace or Docs Live planning workflow.

## Collection Steps

1. Build the release binary from a clean checkout.
2. Run a 30 minute or longer profiling session.
3. Record scenario timings, memory trend, CPU trend, and any UI stalls.
4. Hash profiler output files rather than committing large raw traces.
5. Fill the native profile template generated by
   `pnpm run check:performance-profile`.
6. Validate with:

```sh
NEDITOR_PERFORMANCE_PROFILE_EVIDENCE=/path/to/native-profile.json pnpm run check:performance-profile
```

## Acceptance Criteria

- Evidence names the release device and platform.
- Evidence matches the current app version and Git commit.
- No sustained growth or responsiveness blocker remains unexplained.
- The validator passes after evidence is supplied.
